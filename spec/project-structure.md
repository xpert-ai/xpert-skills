# Project Structure

The repository is organized into three top-level content areas.

```text
xpert-skills/
  skills/
  spec/
  template/
```

## Directory Purposes

### `skills/`
Contains concrete, ready-to-use example skills grouped by category.

### `spec/`
Contains repository-level rules for naming, formatting, and layout.

### `template/`
Contains starter files that can be copied to create new skills quickly.

## Naming Conventions
- Use kebab-case for skill folder names.
- The skill folder name should normally match the `name` field in `SKILL.md`.
- Group skills under stable category folders.

## Recommended Category Layout

```text
skills/
  creative-design/
  development-technical/
  enterprise-communication/
  document-skills/
```
